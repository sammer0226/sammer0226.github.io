NLP中的词向量演变

无标签
Markdown
发布日期:   2020-05-18

思维导图

统计概率模型

  • 知识脉络：语言模型->神经语言模型->word2vec->glove->elmo->bert

  • 如何量化评估一个句子的合理性：用单词序列的联合概率来刻画

  • 假设有两句话

    我今天想要出去玩 VS 我天今要想去出玩。

    我们可以很容易判断出来第一句话更符合我们的语言习惯。深入思考，为什么我们觉得第一句话更像是一句话呢，因为我们会下意识的先对句子做分词，将句子分成 我/今天/想要/出去/玩，而第二句话却很难做分词，因为无论怎么组合，都很难拼凑成我们常见的句子。从这个例子我们不难看出，我们判断一句话是否合理，第一是看构成句子中的字是否能构成几个合理的词语。

    让我们再看另外一个例子。

    南京市长江大桥

    这是很容易引起歧义的一句话，有2种解释方式，一种是南京市/长江大桥，另一种是南京市长/江大桥。

    因此，想要知道这句话到底想表达什么意思，我们还需要知道上下文环境，比如说的如果是南京市长江大桥建成于xxxx，那么建成这个词更经常出现在长江大桥之后，那么我们可以认为这句话大概率是在讲长江大桥，如果上下文环境是南京市长江大桥出席了xxxxx会议。那么我们就会知道出席这个词前面大概率是个人，因此应该把句子划分成南京/市长/江大桥。可见，我们不仅关心字与字之间构成词语的概率，也关心词语之间顺序关系的合理性

    那么，我们如何将直观的感受建模成数学模型和量化指标呢，我们既关心词语的合理性，同时也关心词语之间的共现关系，用数学模型来表示的话，就是一句话出现的概率。而一句话又是由不同的词语序列构成的，因此T位置的词语会受到前T-1个词语的影响，这是一个条件概率。我们引入一个句子的联合概率来表征语言模型

  • 之后的很多无监督预训练词向量的模型都是建立在语言模型之上，我们通过最大化一个句子的联合概率来调整模型参数。其中，对于语言模型，可以有一些前提假设和扩展。比如P(wn|w1,w2…wn-1),我们可以用马尔科夫假设来简化计算量，即我们认为第N位置的词只与前面有限个词有关，而不是和前N-1个词均有关。特别的，当我们假设第N个词只与前一个词有关，那么就是Bi-Gram模型，同样可以类比得到Tri-Gram等。这便是n-gram模型，利用马尔科夫假设来简化计算和缩小共现词表库的大小。因此，在Bi-Gram模型下

    P(wn|w1,w2…wn-1) = P(wn|wn-1)。 此外，我们不仅可以用前N-1个词来限制第N个词，我们还可以用窗口词来刻画该词的上下文环境。然后同样是希望在给定上下文环境时，目标词的出现概率最大。通过这个优化目标来调整参数

    有了优化目标之后，我们首先需要考虑一个问题。一句话如何用数字来表示呢？

    我们在做数据挖掘时，可以输入离散或者连续型变量作为特征，做图像处理时我们输入一张图片，其内部每个像素点用RGB的256种亮度值来表示。而在文本中，我们需要表示一句话，最朴素的思想就是用One-Hot将整个词表大小作为特征维度，如果一句话的某个词出现在这个词表中，则该位置计为1，反之为0。

    举例说明: 例如我们一共有3句话

    1. 今天/天气/真好
    2. 我们/今天/去/学习
    3. 学习/使/我/快乐

    经过分词后，我们统计得到一共有[今天，天气，真好，我们，去，学习，使，我，快乐]一共9个不同的词。那么如果是用One-Hot的方式来表征的话，3句话对应的词向量就应该是

    今天 天气 真好 我们 去 学习 使 我 快乐
    今天天气真好 1 1 1 0 0 0 0 0 0
    我们今天去学习 1 0 0 1 1 1 0 0 0
    学习使我快乐 0 0 0 0 0 1 1 1 1

    词语的语义包含哪些方面

    • 同义词 (Synonyms)： couch/sofa, car/automobile

    • 反义词 (Antonyms)： long/short, big/little

    • 词相似性 (Word Similarity)：cat 和 dog 不是同义词，但是它们是相似的。

    • 词相关性 (Word Relatedness)：词和词之前可以是相关的，但不是相似的。coffee 和 cup 是不相似的，但是它们显然是相关的。

      • 语义域/主题模型(semantic field/topic models LDA)： 一些词同属于某个 semantic domain，相互之间有强相关性，restaurant(waiter, menu, plate, food, chef), house(door, roof, kitchen, family, bed)

    • 语义框架和角色(Semantic Frames and Roles)： 一些词同属于某个事件的角色，buy, sell, pay 属于一次购买的不同角色。

      • 上/下位词(hypernym/hyponym)：一个词是另一个词的父类叫上位词(hypernym)，反之叫下位词(hyponym), vehicle/car, mammal/dog, fruit/mango。

    • 情感/情绪/观点/评价词(connotations, emotions, sentiment, opinions)：正负向情绪词happy/sad，正反向评价词(great, love)/(terrible, hate)

    PMI

    (点互信息): 刻画两个词之间是否强相关

    计算语料库中两个词共同出现的次数是否多于我们先验期望它们偶然出现的概率

    神经网络语言模型

    Word2Vec

    skip-gram: 用中间词预测上下文(更常用，训练更慢，但是效果更好，对于生僻词的识别更友好)

    CBOW：用上下文词预测中间词

    CBOW训练速度要比skip-gram快，原因在于一个滑动窗口内只能产生一个sample，而skip-gram可以产生多个。比如滑动窗口含“the man loves his son”， loves为中心词。对于CBOW，只有一个样本，该样本以loves为中心词，“the man his son”作为背景词；对于skip-gram，有多个样本，即<love, the>, <love, man>, <love, his>, <love, son>。

    skip-gram效果更好一些，尤其是生僻字。原因有两个，一是上一段提到的样本的原因。love在skip-gram有四个sample，可以更新四次参数（假设使用随机梯度下降）。而使用CBOW时”the man his son”只有一个样本，更新一次参数。此外，CBOW用所有背景词的均值来表示背景词信息带入运算，也会损失精度。

    CBOW训练的时候是中心词的词向量不变，更新上下文词向量的梯度，skip-gram反之

预训练思想(2种方式):

  • 一种是 Frozen，就是 Word Embedding 那层网络参数固定不动

    • 另外一种是 Fine-Tuning，就是 Word Embedding 这层参数用现成的词向量初始化，然后参与模型的训练过程
